# Bifrost

## 简介

基于 [websocket库](github.com/gorilla/websocket) 开发的独立消息中心服务

具备以下特点：

* 支持通过jwt claims来作为Client用户内容
* 支持通过指定jwt claims的字段作为Client唯一键
* 可以通过HTTP或RPC（使用Bifrost sdk）调用接口推送消息至用户
* 配置文件可以使用本地YAML或Nacos进行参数设置

## 设计理念

在以往的项目中，每次都需要单独实现 WebSocket 推送功能，这会导致：

* 不可避免地重复复制或重写已有 Hub/工具代码
* 维护成本高，改动容易影响业务系统
* 业务系统代码被推送逻辑侵入

为了解决这些问题，本项目将消息推送功能 独立为统一服务，实现“消息大一统”：

* 业务系统只需调用服务接口，无需关心推送成功与否
* 减少重复开发，降低维护成本
* 提升系统可复用性和一致性
